哈希表

哈希函数

特征

  1. 一致性
  2. 将不同的字符串映射为不同的数值

组成

  • Hash函数 + 数组
  • 数组直接映射到内存
  • Hash Table具有额外逻辑，它使用Hash函数智能的找到存放元素的位置

冲突

原因

两个key的索引相同

解决方法

  • 链接地址法：将索引相同的key以链表形式组织

避免冲突

装载因子

  • 使用的索引个数 / 总共的索引个数

  • 调整大小，Resize

    • 装载因子为75%的时候，可以调整大小，通常为原本大小的两倍
    • 调整大小也会花费很多时间

插入

命令：put(key, value)

  1. index = hash(key)
  2. insertNode(index, key, value)

存在冲突：链接地址法，如果key相同，则直接修改value

查询

命令：get(key)

  1. index = hash(key)
  2. searchNode(index, key)

当链表过长，将链表转换为红黑树

链表长度 > 8，转换为红黑树

红黑树深度 < 6，转换为链表

扩容

  1. 当发现HashMap内元素过多时，需要扩容
  2. 创建一个新的数组，大小为之前的两倍，即 $2^n$
  3. 遍历旧数组上链表的结点，按照数组大小重新计算index
  4. 按照头插法，把旧数组上的节点插入新的数组里，用新的数组+链表代替旧的即可

知识点

  • HashMap默认长度为16，HashMap长度为 $2^n$
  • 线程不安全，如果两个线程同时触发扩容操作，就可能出现线程安全问题
